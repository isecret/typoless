# Typoless Epics And Stories

## 1. 文档目的

本文档将 [PRD.md](./PRD.md) 中已冻结的产品需求拆分为可执行的 `Epics` 和 `Stories`，用于后续排期、任务分派和 issue 管理。

拆分原则：

- 以首版 `MVP` 为边界
- 优先保证主链路可用
- 每个 Story 必须具备明确交付结果
- 每个 Story 尽量可独立验收

## 3. Epic 详情

## E1. 应用骨架与菜单栏

### 目标

建立首版 macOS 菜单栏应用骨架，提供最基础的应用入口和状态展示能力。

### Stories

#### S1.1 初始化 macOS 应用工程

作为开发者，我需要一个基于 `Swift + SwiftUI + AppKit` 的项目骨架，以便后续实现菜单栏应用。

验收标准：

- 工程可在 Xcode 中打开并运行
- 应用启动后可正常驻留于菜单栏
- 项目目录结构清晰，至少包含 UI、Domain、Providers、Platform、Persistence 分层入口
- 客户端整体架构采用 `MVVM + Service Layer`

#### S1.2 实现菜单栏图标与菜单

作为用户，我希望应用在菜单栏中有基础入口，以便访问核心功能。

验收标准：

- 菜单栏图标可见
- 菜单中包含 `打开设置`、`退出应用`
- 处理中状态下菜单中出现 `取消当前任务`
- 注入失败时菜单中显示失败文本预览，可点击复制

#### S1.3 实现状态驱动的菜单栏反馈

作为用户，我希望菜单栏图标能表达当前处理状态，以便知道应用正在做什么。

验收标准：

- 支持显示 `空闲`、`录音中`、`识别中`、`润色中`、`注入中`、`失败`
- 状态变化能随主链路正确刷新

依赖：

- 无

## E2. 设置页与本地配置

### 目标

提供首版必需的配置入口，并实现普通配置与密钥的本地持久化。

### Stories

#### S2.1 实现设置页基础框架

作为用户，我希望有一个统一的设置页配置应用，以便完成首次初始化。

验收标准：

- 应用可从菜单打开设置页
- 首次启动自动打开设置页
- 设置页包含 LLM、快捷键、权限、诊断模块占位

#### ~~S2.2 实现腾讯云 ASR 配置表单~~

> 已移除：ASR 方案已收敛为本地 Whisper，不再需要腾讯云配置表单。

#### S2.3 实现 OpenAI 兼容 LLM 配置表单

作为用户，我希望填写 LLM 配置，以便应用可以完成文本润色。

验收标准：

- 支持输入 `Base URL`、`API Key`、`Model`
- 配置值可保存并重新加载

#### S2.4 实现功能开关与快捷键配置项

作为用户，我希望设置全局快捷键并控制是否启用 AI 润色。

验收标准：

- 支持设置并展示全局快捷键
- 支持 `启用 AI 润色` 开关
- 配置值可保存并重新加载

#### S2.5 实现配置存储策略

作为开发者，我需要区分普通设置和敏感密钥的存储位置，以保证安全性和可维护性。

验收标准：

- LLM API Key 存储到 `~/.typoless/config`
- 普通设置存储到 `UserDefaults`
- 应用重启后可恢复配置
- 保存配置时做轻量格式校验，真正联网调用时再做严格失败

依赖：

- E1

## E3. 权限管理

### 目标

完成首版所需的麦克风权限与辅助功能权限引导和状态展示。

### Stories

#### S3.1 实现麦克风权限检测与引导

作为用户，我希望知道应用是否有录音权限，并能被引导去授权。

验收标准：

- 设置页可展示麦克风权限状态
- 权限缺失时提供明确引导
- 未授权时无法开始录音

#### S3.2 实现辅助功能权限检测与引导

作为用户，我希望知道应用是否有文本注入权限，并能被引导去授权。

验收标准：

- 设置页可展示辅助功能权限状态
- 权限缺失时提供明确引导
- 未授权时注入动作会失败并返回明确错误

#### S3.3 实现首次配置权限流程

作为用户，我希望首次打开设置页时能按顺序完成权限准备，而不是在主流程里被打断。

验收标准：

- 首次启动自动打开设置页
- 设置页中权限区展示引导顺序清晰
- 权限申请时机与设置页流程一致

依赖：

- E1
- E2

## E4. 全局快捷键与录音

### 目标

打通首版录音能力，支持全局快捷键按下切换模式触发录音。

### Stories

#### S4.1 实现全局快捷键注册

作为用户，我希望通过全局快捷键触发录音，而不必切换到应用窗口。

验收标准：

- 应用可注册单一全局快捷键
- 快捷键在常驻菜单栏状态下可生效
- 修改快捷键后立即生效

#### S4.2 实现按下切换录音模式

作为用户，我希望按一次快捷键开始录音、再按一次结束录音，以获得简洁直觉的交互。

验收标准：

- 按下快捷键开始录音
- 再次按下快捷键结束录音
- 处理中（`transcribing / polishing / injecting`）再次按键忽略
- 录音状态能反映到菜单栏

#### S4.3 实现录音时长与超时控制

作为产品，我需要限制单次录音长度，以控制延迟和成本。

验收标准：

- 单次录音上限为 `30 秒`
- 超时后自动结束录音
- 超时结束后仍继续主链路处理

#### S4.4 实现音频格式标准化

作为开发者，我需要统一录音输出格式，以便稳定提交给 ASR。

验收标准：

- 音频输出格式固定为 `PCM/WAV 16k mono`
- 主链路可复用标准化后的音频结果

> 已废弃：需求变更为仅保留按下切换模式，不再支持双模式切换。见 S4.2。

## E6. OpenAI 兼容 LLM 集成

### 目标

接入用户自定义 OpenAI 兼容模型服务，实现固定边界的文本润色。

### Stories

#### S6.1 实现 OpenAI 兼容 LLM Provider

作为开发者，我需要一个标准化的 LLM 调用层，以便对接不同 OpenAI 兼容网关。

验收标准：

- 对齐 `OpenAI Chat Completions` 接口形态
- 支持 `Base URL`、`API Key`、`Model`
- 调用层独立于 UI
- 可返回统一的文本结果模型
- Provider 内部负责固定超时与取消控制

#### S6.2 实现固定润色 Prompt 策略

作为产品，我希望首版的 LLM 行为边界稳定可控。

验收标准：

- Prompt 明确限制为纠错、去赘词、轻度书面化、补中文标点
- 不暴露用户自定义 Prompt 能力
- 不暴露高级模型参数

#### S6.3 实现 AI 润色开关

作为用户，我希望在需要时关闭 LLM 润色，直接输出 ASR 原文。

验收标准：

- 设置页中的 `启用 AI 润色` 可控制是否调用 LLM
- 关闭时主链路直接进入文本注入

#### S6.4 实现 LLM 失败回退

作为用户，我希望即使 LLM 出错，也能尽量得到可用文本结果。

验收标准：

- LLM 超时、失败、空结果时自动回退 ASR 原文
- 回退结果可正常进入注入
- 记录状态标记为 `fallback_success`

依赖：

- E2

## E7. 文本注入与回退

### 目标

把最终结果稳定写入当前焦点应用，并在失败时提供最小可用回退。

### Stories

#### S7.1 实现文本注入能力

作为用户，我希望识别结果能直接进入当前输入框，而不是停留在应用内部。

验收标准：

- 优先通过焦点元素 `AX` 写值注入文本
- `AX` 写值失败时回退到键盘事件输入
- 适配常见场景：浏览器输入框、备忘录、聊天应用

#### S7.2 实现注入失败处理

作为用户，我希望注入失败时仍然能拿到结果文本。

验收标准：

- 注入失败时返回用户可理解错误
- 不自动覆盖剪贴板
- 失败文本保留在内存中，菜单栏显示截断预览，点击可复制到剪贴板
- 该失败文本仅在当前运行期有效，下一次成功注入后自动清空

#### S7.3 实现辅助功能权限校验接入注入链路

作为系统，我需要在注入前确认权限满足条件，以避免无意义失败。

验收标准：

- 注入前进行辅助功能权限检查
- 缺权限时终止注入并提示

依赖：

- E3
- E6

## E8. 状态机、错误处理与取消

### 目标

保证主链路稳定串行运行，并支持取消、错误分类和状态驱动 UI。

### Stories

#### S8.1 实现 Session Coordinator

作为开发者，我需要统一的主链路协调器管理状态和依赖，以避免流程散落各处。

验收标准：

- 串行控制 `idle -> recording -> transcribing -> polishing -> injecting -> done`
- 支持 `error` 和 `cancelled` 异常状态
- 同一时间只允许一个 session

#### S8.2 实现用户可理解的错误分类

作为用户，我希望看到简明错误，而不是底层异常信息。

验收标准：

- 覆盖麦克风权限、辅助功能权限、本地识别失败、LLM 凭证或模型、LLM 网络、注入失败、用户取消
- 错误可用于菜单栏和设置页摘要展示

#### S8.3 实现处理中取消能力

作为用户，我希望识别或润色卡住时可以主动终止当前任务。

验收标准：

- 识别中和润色中可从菜单栏取消
- 取消后请求停止或结果被丢弃
- 取消后不进入文本注入

依赖：

- E4
- E6
- E7

## E10. 集成验证与发布准备

### 目标

完成首版 MVP 的联调、验收和交付准备。

### Stories

#### S10.1 完成主链路端到端联调

作为团队，我需要确认从录音到注入的完整链路可用。

验收标准：

- 在真实配置下完成录音 -> ASR -> LLM -> 注入

作为团队，我需要最小可维护文档，支持后续开发和验收。

验收标准：

## 5. MVP 完成定义

以下条件全部满足，视为首版 MVP 完成：

- 用户可完成首次配置和权限准备
- 用户可通过全局快捷键完成一次中文短语音输入
- 本地 Whisper 可返回有效转写
- OpenAI 兼容 LLM 可完成固定边界内的文本润色
- LLM 失败时可自动回退到 ASR 原文
- 最终文本可注入常见 macOS 应用输入区域
- 关键错误可被用户理解和处理
- 注入失败时文本可通过菜单栏临时复制入口恢复
- 技术设计与实现约束已沉淀到 [TDD.md](./TDD.md)

# Typoless PRD

## 1. 文档信息

- 产品名称：Typoless
- 文档类型：PRD
- 版本：v1.2
- 状态：已更新
- 更新时间：2026-04-29

## 2. 产品概述

Typoless 是一款面向 macOS 的语音 + AI 输入助手。用户通过全局快捷键触发录音（按一次开始，再按一次结束），录音结束后自动完成语音识别、文本润色，并将结果写回当前焦点应用。

首版产品不是系统级输入法，而是菜单栏常驻应用。产品重点是用最短路径实现“语音输入 + AI 整理”的稳定闭环，降低系统输入法开发复杂度，同时保留未来升级为真正输入法的架构空间。

## 3. 目标与非目标

### 3.1 产品目标

- 在 macOS 上提供可常驻运行的全局语音输入能力。
- 首版优先优化中文短语音输入体验。
- 使用本地离线 ASR（基于 `FunASR`）或腾讯云一句话识别完成语音转写。本地模型外置存储于用户目录 `~/.typoless/models/funasr/`，通过设置页引导下载。
- 用户可在设置中手动选择 ASR 平台（本地 FunASR 或腾讯云一句话识别）。
- 通过本地降噪、个人词典与 AI 润色提升中文短语音识别落地质量。
- 接入用户自定义的 OpenAI 兼容大模型服务完成文本纠错与轻度润色。
- 在常见桌面应用中稳定完成文本注入。

### 3.2 非目标

- 不实现 macOS 系统级 Input Method。
- 不支持多种 LLM 协议，仅支持 OpenAI 兼容接口。
- 不支持自定义提示词、风格模式、temperature、max tokens 等高级参数。
- 不支持保存音频历史。
- 不做 agent 命令执行、翻译工作流或复杂生产力动作。
- 不做 ASR 平台自动回退；当前选中平台不可用时录音前直接阻止。

## 4. 目标用户

- 有中文语音输入需求的 macOS 用户。
- 希望将口语内容快速整理成可直接发送或写入的文本的用户。
- 有自有大模型服务，或愿意使用 OpenAI 兼容网关服务的用户。

## 5. 核心使用场景

- 在聊天工具中通过语音快速输入较自然、可直接发送的中文句子。
- 在笔记类应用中用语音输入段落，并自动完成基础纠错和标点整理。
- 在浏览器表单、搜索框、输入框中通过语音替代部分键盘输入。

## 6. 核心用户流程

### 6.1 首次使用流程

1. 用户启动应用。
2. 应用内置降噪、FunASR runtime 与 worker；若使用本地 FunASR，用户需先在设置页下载模型。
3. 用户配置 LLM：
   - `Base URL`
   - `API Key`
   - `Model`
4. 用户设置全局快捷键。
5. 用户在设置页中完成麦克风权限授权。
6. 用户在设置页中完成辅助功能权限授权。
7. 应用进入菜单栏常驻状态。

### 6.2 日常使用流程

1. 用户在任意应用中聚焦输入区域。
2. 用户按下全局快捷键开始录音。
3. 用户再次按下快捷键结束录音（或达到 60 秒自动结束）。
4. 客户端对本次音频进行本地降噪处理。
5. 录音结束后，本地离线 ASR 返回转写文本。
6. 客户端调用 OpenAI 兼容接口进行文本处理。
7. 仅当 LLM 成功返回可用结果时，客户端才进入文本注入。
8. 客户端将最终文本一次性写入当前焦点应用。
9. 若 LLM 配置不完整或请求失败，则直接提示错误，不注入任何文本。
10. 菜单栏状态恢复空闲。

## 7. 产品形态与交互

### 7.1 应用形态

- 菜单栏常驻应用。
- 无主编辑窗口。
- 设置页为唯一完整配置界面。

### 7.2 快捷键交互

- 首版仅支持单一全局快捷键。
- 录音触发方式固定为按下切换：按一次开始录音，再按一次结束录音。
- 在 `识别中 / 润色中 / 注入中` 阶段再次按快捷键时忽略，不作为取消入口。
- 不支持多快捷键动作体系。

### 7.3 状态反馈

菜单栏图标需表达以下状态：

- 空闲
- 录音中
- 识别中
- 润色中
- 注入中
- 失败

系统通知用于提示：

- 配置错误
- 权限缺失
- 本次处理失败
- 注入失败

### 7.4 处理中断

- 当应用处于识别中或润色中时，用户可从菜单栏取消当前任务。
- 取消后不再注入文本。

## 8. 功能需求

### 8.1 菜单栏

菜单栏需包含以下能力：

- 展示当前状态
- 打开设置页
- 当前错误摘要（存在时显示）
- 注入失败文本预览（仅当最近一次注入失败且文本存在时显示，点击复制到剪贴板）
- 取消当前任务（仅处理中显示）
- 退出应用

### 8.2 设置页

设置页需包含以下模块：

#### A. LLM 配置

- Base URL 输入框
- API Key 输入框
- Model 输入框

#### B. 快捷键配置

- 全局快捷键设置入口

#### C. 权限状态

- 麦克风权限状态
- 麦克风权限引导
- 辅助功能权限状态
- 辅助功能权限引导

#### D. 诊断信息

- 最近一次错误摘要

#### E. 个人词典

- 词条新增、删除、启用、禁用入口
- 词条至少包含术语文本
- 可选填写发音提示和分类

#### F. ASR 配置

- ASR 平台选择（本地 `FunASR` / 腾讯云一句话识别）
- 本地模型状态展示与下载入口
- 下载中显示进度与取消
- 已就绪时仅显示完成态，不暴露模型路径、模型版本、重新下载或删除入口

### 8.3 录音能力

- 录音触发方式固定为按下切换：按一次快捷键开始录音，再按一次结束录音。
- 单次录音最长 60 秒。
- 超过 60 秒自动结束录音并继续处理。
- 录音时长低于 500ms 时视为误触，静默取消，不进入降噪、ASR、LLM 或文本注入。
- 短录音静默取消不展示错误摘要，不生成注入失败文本，仅允许写入内部诊断日志。
- 若麦克风权限缺失，则不能开始录音，并给出明确提示。

### 8.4 音频预处理与语音识别

- 录音结束后、进入 ASR 前，默认执行本地降噪处理。
- 降噪方案使用开源 RNNoise。
- 默认 ASR 方案为本地 `FunASR` 离线识别，通过内置 Python sidecar 运行，使用 paraformer-zh + fsmn-vad 固定模型组合。
- 录音结束后取 ASR 转写文本进入 LLM 润色与注入流程。
- App 仅内置降噪、FunASR runtime、worker 和 manifest；FunASR 模型必须外置到用户目录，缺失时阻止录音并给出明确提示。
- 当用户选择本地 `FunASR` 时，设置页仅展示模型状态和下载/取消等必要操作，不向用户暴露模型路径或模型版本。
- 旧 Whisper 和 sherpa-onnx 链路可作为历史实现保留，但不作为默认 ASR 路径，不作为资源缺失时的自动回退，也不再作为正式包内置资源分发。
- 本轮不暴露 ASR 高级参数（模型选择、线程数、热词权重等）。

### 8.5 LLM 润色

- 首版唯一 LLM 协议为 OpenAI 兼容接口。
- 用户仅可配置 `Base URL`、`API Key`、`Model`。
- 不暴露 temperature、max tokens、system prompt 等高级设置。
- Prompt 固定由应用内置维护，不向用户开放自定义 Prompt。
- 个人词典可作为术语参考进入 Prompt，但不得被视为可执行指令。

LLM 文本处理边界如下：

- 允许：
  - 修正 ASR 识别错误
  - 修正常见同音词和错别字
  - 去掉明显口语赘词
  - 轻度书面化表达
  - 自动补自然中文标点
  - 保留个人词典中的专有名词
- 不允许：
  - 大幅改写原句结构
  - 主动扩写未说出的内容
  - 擅自改变语义、语气、事实

### 8.6 输出策略

- 默认输出润色版。
- 仅当 `Base URL`、`API Key`、`Model` 三项均完整时，才视为 LLM 已配置完成。
- 若 LLM 配置不完整，则本次流程直接报错，不注入任何文本。
- 若 LLM 失败、超时、返回无效内容或明显为空，则本次流程直接报错，不注入任何文本。

### 8.7 文本注入

- 最终文本通过 macOS Accessibility API 与输入事件写回当前焦点应用。
- 若辅助功能权限缺失，则注入失败并提示用户授权。
- 注入失败时不自动写入剪贴板。
- 注入失败的文本保留在内存中，菜单栏显示截断预览，用户点击可复制完整文本到剪贴板。
- 该失败文本仅在当前应用运行期间有效，不落盘。
- 下一次成功注入后自动清空该临时失败文本。

### 8.8 注入失败恢复

当文本注入失败时，菜单栏提供最小恢复路径：

- 菜单栏第一行显示当前错误摘要
- 当且仅当存在最近一次注入失败文本时，菜单栏第二行显示该文本的截断预览（约 20 个字符加省略号）
- 点击预览可复制完整文本到系统剪贴板
- 该失败文本仅在当前应用运行期间有效，不落盘
- 成功完成一次新的文本注入后清空该临时失败文本
- 仅文本注入失败会生成该失败文本入口；取消、ASR 失败、权限失败、LLM 失败均不会生成

## 9. 系统状态机

应用处理状态机固定为：

`idle -> recording -> transcribing -> polishing -> injecting -> done`

异常路径补充状态：

- `error`
- `cancelled`

状态规则：

- 同一时间仅允许一个 session。
- 新 session 不得覆盖正在处理中的旧 session。
- 用户取消后当前 session 立即终止。

## 10. 配置项定义

首版公开配置字段固定如下：

- `openai_base_url`
- `openai_api_key`
- `openai_model`
- `global_hotkey`
- `pasteboard_injection_bundle_ids`

存储规则：

- 全部配置（含密钥）统一存储于 `~/.typoless/config.json`（UTF-8 JSON）。
- 首次升级时自动从旧存储（UserDefaults + Keychain）迁移。
- 个人词典存储于 `~/.typoless/dictionary.json`，仅保存用户显式维护的术语，不保存历史输入文本。

## 11. 错误反馈策略

错误对用户只展示简明、可理解的分类文案，不直接展示底层原始报文。

首版需要覆盖的错误类别：

- 麦克风权限未开启
- 辅助功能权限未开启
- 录音数据为空
- 本地识别引擎未就绪
- 本地识别模型缺失
- 本地降噪资源缺失或处理失败
- 本地语音识别失败
- LLM 凭证或模型无效
- LLM 网络异常
- 文本注入失败
- 当前任务已取消

错误展示位置：

- 菜单栏状态
- 系统通知
- 设置页最近一次错误摘要

## 12. 非功能需求

### 12.1 稳定性

- 连续多次触发录音时，不得出现状态串扰或旧结果覆盖新结果。
- 取消后的请求不得继续进入注入流程。

### 12.2 隐私

- 默认不保存音频。
- 不保存历史文本记录。
- 语音识别在本地完成，音频数据不离开设备。
- 明确提示用户：文本会发送至用户配置的 LLM 云端服务。
- Release 构建日志不得输出原始音频、ASR 原文、LLM 输出正文或用户密钥。
- Debug 构建允许输出 ASR/LLM 明文对照日志，用于开发排查，不作为用户数据持久化能力。

### 12.3 可扩展性

- ASR 层保留替换识别引擎的架构空间。
- LLM Provider 层需与 UI 解耦。
- 当前架构应支持未来升级为真正输入法或扩展更多实时反馈能力。

## 13. 技术约束

- 客户端技术栈：`Swift + SwiftUI + AppKit`
- 使用菜单栏应用架构
- 使用 macOS Accessibility API 做文本注入
- 不依赖系统级输入法框架实现首版能力

## 14. 验收标准

### 14.1 基础可用

- 首次启动自动打开设置页。
- 用户可完成 LLM、快捷键、权限配置。
- 应用可常驻菜单栏并正常响应全局快捷键。

### 14.2 主链路可用

- 用户可通过快捷键触发录音并完成一次短语音输入。
- 本地 `FunASR` 可返回中文转写。
- 降噪链路可正常处理录音音频。
- LLM 成功时输出润色版文本。
- LLM 配置不完整或请求失败时不会注入任何文本。
- 文本可在常见应用中成功注入。

### 14.3 失败场景可控

- 缺少麦克风权限时无法录音且提示明确。
- 缺少辅助功能权限时注入失败且提示明确。
- LLM 配置错误时，用户能看到可理解的错误摘要。
- 本地识别失败时，用户能看到可理解的错误信息。
- 注入失败后，用户仍可从菜单栏复制失败文本。

### 14.4 数据行为正确

- 注入失败文本仅在当前运行期有效。
- 成功注入后失败文本自动清空。
- 应用重启后保留配置，不保留音频和历史文本。

## 15. 版本边界

### 15.1 v1 必做

- 菜单栏应用
- 设置页
- 本地 `FunASR` 语音识别
- RNNoise 本地降噪
- OpenAI 兼容 LLM 润色
- 全局快捷键
- 文本注入
- 权限引导
- 注入失败文本临时复制入口
- 失败回退
- 个人词典
- 诊断耗时日志与 Debug ASR/LLM 对照日志

### 15.2 v1 不做

- 系统输入法实现
- ASR Provider 自动回退
- 云端 ASR 回退
- 自定义 Prompt
- 风格模式切换
- 高级模型参数
- ASR 高级运行参数
- 音频历史
- 自动复制剪贴板兜底

## 16. 结论

首版 Typoless 的产品定义为：一款运行于 macOS 菜单栏的中文语音 + AI 输入助手，基于本地降噪、`FunASR` 本地语音识别、个人词典与用户自定义 OpenAI 兼容 LLM 服务，提供“快捷键触发录音，自动处理，直接注入文本”的稳定输入体验。
